Welcome to CSS Style Guide

Follow the steps to understand CSS style structure

Step 1: Variables.scss
Definition
Variables are used to store information that can be reused throughout your stylesheet. They make it easy to update your styles and keep your code clean and organized.
How to use
Variables are defined using the $ symbol followed by the variable name and a value
Example
$primary-color: #007bff; defines a variable named primary-color with a value of #007bff
Naviagation Path
src > app > styles > variables.scss
Properties
  • Color: font-color, backgropund-color, border-coor
  • Font: font-family, font-size, font-weight
  • Spacing: padding, margin
  • Border: border-radius, border-width, border-color
  • Shadow: box-shadow, text-shadow
 Step 2: Typography.scss
Definition
It is used to define all the HTML semantic element styles
Navigation Path
src > app > styles > typography.scss
Properties
H1 to H6, P, span, buttons, anchor
 Step 3: Layouts.scss
Definition
It is used to define the layout & components of the page
Navigation Path
src > app > styles > layouts.scss
Properties
  • Flex: flex-direction, justify-content, align-items
  • Grid: grid-template-columns, grid-template-rows, grid-gap
  • Header: Title, Subtitle, headerBar
  • Card: card-container, card-header, individual-card
 Step 4: Mixins.scss
Definition
Mixins are used to define reusable styles that can be applied to multiple elements. They make it easy to keep your code DRY and maintainable.
How to use
Mixins are defined using the @mixin directive followed by the mixin name and a block of styles
Navigation Path
src > app > styles > mixins.scss
Properties
  • Button: Primary, seconadary, disabled
  • Flex: Direction, align, wrap
  • Grid: grid-templpate-columns, grid-template-rows, gap,
Step 5: Helpers.scss
Definition
Helpers are used to define utility classes that can be applied to elements to achieve specific styles or behaviors. They make it easy to add common styles to your elements without having to write custom CSS.
How to use
Helpers are defined using the .class-name syntax followed by a block of styles
Navigation Path
src > app > styles > helpers.scss
Properties
  • Spacing: padding, margin
  • Visibility: hidden, visible
